# 第八章：UI/UX 设计

## 8.1 设计原则

### 视觉原则
- **简约克制**：与游戏的几何美术风格保持一致
- **信息清晰**：关键信息一目了然，不干扰战斗
- **层次分明**：重要信息突出，次要信息收敛
- **动态反馈**：状态变化有明显视觉反馈

### 交互原则
- **响应迅速**：UI 操作即时响应
- **操作简单**：减少点击层级
- **容错设计**：关键操作有确认
- **手感舒适**：动画流畅，过渡自然

### HUD 元素

#### 顶部信息栏
| 元素 | 位置 | 说明 |
|------|------|------|
| HP 条 | 左上 | 当前生命/最大生命，低于 30% 变红 |
| 波次 | 中上 | 当前波次数 |

#### 底部技能栏
| 元素 | 位置 | 说明 |
|------|------|------|
| 技能图标 | 底部居中 | 显示绑定按键 |
| 冷却显示 | 图标上 | 数字倒计时或填充动画 |
| 闪避按钮 | 底部右侧 | 显示 SHIFT 提示 |

#### 战斗信息
| 元素 | 位置 | 说明 |
|------|------|------|
| 连击数 | 左下 | 当前连击，高连击时放大 |
| 伤害数字 | 敌人位置 | 浮动显示，暴击放大 |
| 波次预告 | 中央 | 新波次开始时显示 |

### 视觉反馈
| 状态 | 反馈 |
|------|------|
| 受伤 | 屏幕边缘红色闪烁 |
| 暴击 | 伤害数字放大 + 黄色 |
| 技能就绪 | 图标发光提示 |
| 无双状态 | 屏幕边缘金色光效 |
| 濒死 | HP 条闪烁 + 心跳音效 |

- 悬停：卡片放大 + 详细说明
- 选中：卡片发光 + 确认动画

| 史诗 | 紫色 | 深紫 | 粒子效果 |

## 8.8 UI 动画规范

### 过渡动画
| 场景 | 动画 | 时长 |
|------|------|------|
| 界面切换 | 淡入淡出 | 300ms |
| 按钮悬停 | 放大 1.05x | 150ms |
| 按钮点击 | 缩小 0.95x | 100ms |
| 面板弹出 | 从中心放大 | 200ms |

### 战斗动画
| 元素 | 动画 | 说明 |
|------|------|------|
| 伤害数字 | 向上浮动 + 淡出 | 1s |
| 连击数 | 抖动 + 放大 | 高连击时 |
| HP 变化 | 平滑过渡 | 0.3s |
| 技能冷却 | 顺时针填充 | 冷却时间 |

### 颜色动画
| 状态 | 动画 |
|------|------|
| HP 低 | HP 条红色脉冲 |
| 技能就绪 | 图标呼吸发光 |
| 新波次 | 屏幕边缘光波 |
